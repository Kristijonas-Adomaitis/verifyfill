Purpose

VerifyFill is a Chrome extension that helps you complete website sign-in, account creation, and two-step verification without switching between Gmail and the website you are on.

When a website sends a verification code or confirmation link to your Gmail inbox, VerifyFill reads your recent inbox messages, finds the code or link, and offers to autofill it on the page where you are signing in.

Functionality

VerifyFill does the following:

  • Google Sign-In — connect your Gmail account through the extension popup
  • Code detection — finds OTP verification codes in recent inbox messages
  • Link detection — finds email confirmation and sign-in links in recent messages
  • Autofill prompt — shows a prompt on the page when a matching code or link is found
  • One-click fill — inserts a code into the input field you are using
  • One-click open — opens a detected verification link when you choose
  • Background checks — periodically checks Gmail while you are signed in
  • Manual check — scan inbox on demand from the extension popup
  • Sign out — disconnect Google and clear stored verification history

VerifyFill does not send email, delete email, read your full mailbox, or run a server that stores your inbox content.

How it works

  1. Install the extension in Chrome and sign in with Google.
  2. Visit a website that asks for a verification code or tells you to check your email.
  3. VerifyFill searches recent Gmail inbox messages for a matching code or link.
  4. You click the on-page prompt to autofill the code or open the link.

How VerifyFill uses Google user data

VerifyFill requests Google account access only to identify your connected account and read recent Gmail messages that may contain verification codes or links.

Google account email address

OAuth scope: https://www.googleapis.com/auth/userinfo.email
Why: To confirm sign-in and show which Google account is connected in the extension popup.
What we read: Your Google account email address only.
What we do with it: Display it in the extension. Stored locally on your device while signed in.
What we do not do: Sell it, share it with advertisers, or send it to a VerifyFill server.

Gmail inbox messages (read-only)

OAuth scope: https://www.googleapis.com/auth/gmail.readonly
Why: Websites send verification codes and links to Gmail. VerifyFill needs read-only inbox access to find them. Read-only means we cannot send, delete, or modify email.
What we read:

  • Inbox messages only — not sent mail, drafts, spam, or trash
  • Messages received in the last 15 minutes
  • Up to 20 most recent messages in that time window
  • Subject line, body text, sender address, and message ID for each message

When we read Gmail: After sign-in; about every minute while the extension is active; when you click Check now; when you focus a verification code input or open a check-your-email page.
What we do not do: Upload email to a server, read your full mailbox, use Gmail data for advertising, or access Gmail when you are signed out.

Website page you are using

Permission: Chrome extension access to the active browser tab (not a Google OAuth scope).
Why: To show the autofill prompt and insert the verification code on the page you are using.
What we do not do: Track your browsing history, collect pages for advertising, or send page content to a server.

Local storage on your device

Sign-in state, connected email address, OAuth tokens (managed by Chrome), and IDs of verification codes, links, and messages you already used. This is not a copy of your inbox.

Data we do not request

Google Drive, Calendar, Contacts, Photos, permission to send mail, permission to delete mail, or your Google password.

Your control

You can sign out in the extension, revoke access at Google Account permissions, or uninstall the extension at any time. See the full Privacy Policy.
