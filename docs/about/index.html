About VerifyFill

Application type: Chrome extension · Owner: Kristijonas · kristijonasvr@gmail.com · Privacy Policy

Purpose

VerifyFill is a Chrome extension that helps you finish website sign-in, account creation, and two-step verification without switching between Gmail and the site you are on.

When a service sends a verification code or confirmation link to your Gmail inbox, VerifyFill finds it and offers to autofill the code or open the link on the page where you are signing in.

Functionality

VerifyFill does the following:

  • Google Sign-In Connect your Gmail account through the extension popup.
  • Code detection Finds numeric and alphanumeric OTP codes in recent inbox messages.
  • Link detection Finds email confirmation and sign-in links in recent messages.
  • Autofill prompt Shows an on-page prompt near your input when a match is found.
  • One-click fill Inserts codes into focused fields, including multi-box OTP forms.
  • One-click open Opens detected verification links when you choose.
  • Auto watch Starts watching when you focus a code field or land on a verify-email page.
  • Background checks Periodically checks Gmail while signed in for new codes and links.
  • Check now Manual inbox scan from the extension popup at any time.
  • Used-item memory Does not re-prompt for codes, links, or messages you already used.
  • Sign out Disconnect Google and clear stored verification history.

VerifyFill does not send email, delete email, read your full mailbox, or operate any server that receives your inbox content.

How it works

  1. Install the extension in Chrome and sign in with Google to connect Gmail.
  2. Visit a website that asks for a verification code or tells you to check your email.
  3. VerifyFill searches your recent Gmail inbox for a matching code or link.
  4. A prompt appears on the page — click to autofill the code or open the link.

How VerifyFill uses Google user data

VerifyFill only works if it can identify which Google account you connected, read recent Gmail messages that may contain a verification code or link, and interact with the browser tab where you are signing in.

1. Google account email address

OAuth scope: https://www.googleapis.com/auth/userinfo.email

Used to confirm sign-in and show which account is connected in the extension popup. Stored locally on your device while signed in. Not sold or sent to a VerifyFill server.

2. Gmail inbox messages (read-only)

OAuth scope: https://www.googleapis.com/auth/gmail.readonly

Websites send verification codes and links to Gmail. Read-only access means VerifyFill cannot send, delete, or modify email.

What we read:

  • Inbox messages only — not sent, drafts, spam, or trash
  • Received in the last 15 minutes
  • Up to 20 most recent messages in that window
  • Subject line, body text, sender, and message ID per message

When: After sign-in; about every minute while active; on Check now; when you focus a verification input or land on a check-your-email page.

What we do not do: Upload email to a server, read your full mailbox, use data for ads, or access Gmail when signed out.

3. The website page you are using

Chrome extension tab permission (not a Google OAuth scope). Used to show a fill prompt and autofill the code on the page you are using. We do not track browsing history or send page content to a server.

4. Local storage on your device

Sign-in state, connected email, OAuth tokens (via Chrome), and IDs of used messages, codes, and links. Not a copy of your inbox.

5. Data we do not request

Google Drive, Calendar, Contacts, Photos, send-mail access, delete-mail access, or your Google password.

6. Your control

Sign out in the extension, revoke access in Google Account permissions, or uninstall the extension at any time. Full details: Privacy Policy.
